# Main-Flow Scene UI Checklist

## Quick Searches

先做静态搜索，确认没有明显回退：

```bash
rg -n "font-heading|font-ma-shan-zheng" src/react-app/routes/game src/react-app/components/game-shell src/react-app/components/func
rg -n "InkSection" src/react-app/routes/game src/react-app/components
rg -n "actionBar|sceneSurfaceClassName|sceneBodyClassName|scene[A-Z][A-Za-z]*ClassName" src/react-app
rg -n "variant=\\\"drawer\\\"|onOpenCultivator|WorldChatHost|CultivatorOverviewOverlay" src/react-app
```

如果当前任务只改了某个 feature，也要顺着引用检查它进入主流程正文时依赖的共享组件。

## First-Screen Audit

逐页检查第一屏，只问这些问题：

1. 玩家能否一眼分清自己在哪、现在能做什么、要离开时从哪里走。
2. 玩家是否不用先读完一大段规则，就能明白这个场景是干什么的。
3. 第一屏是否已经被控件本身或场景描述解释清楚；如果是，是否还多了一层解释型标题。
4. 第一屏是否过早摆出了价格、概率、百分比、风险边界或例外说明。
5. 第一屏是否有任何正文元素在争夺场景身份层的优先级。
6. 第一屏是否有跨场景按钮把注意力从当前任务拉走。
7. 第一屏是否有特殊字体、过大字号或强装饰跑进正文。
8. 第一屏是否把角色页、道身页、背包页已有的信息又整块搬了进来。

如果答案不干净，先删冗余结构，再考虑补视觉。

## Layer Audit

检查三个层级是否混用：

- 身份层：是否只有一个强识别中心
- 任务层：是否直接围绕当前场景任务展开
- 导航层：是否被收束在稳定全局入口，而不是散落在正文里

如果同一元素同时承担两个层级职责，通常说明结构需要收口。

## Title Audit

逐个看标题，只问两件事：

1. 它是在命名场景，还是在组织信息。
2. 它有没有新增信息量。

保留：

- 承担流程定位的标题
- 承担信息分组的标题
- 承担结果总结的标题

删除或降级：

- 复述当前 tab、筛选器、列表或表单用途的标题
- 只是把路由名再说一遍的标题
- 只是把规则说明换一种排版再讲一遍的标题
- 通过特殊字体和大字号制造 display 感的正文标题

## Cognitive-Load Audit

检查正文是否把需要后置的信息提前摊开：

- 精确代价是否可以后置到确认弹窗
- 风险提示是否可以后置到确认弹窗
- 数值区间是否真的属于首屏决策前提
- 页面是否为了“完整”展示了不阻断当前动作的状态细项

发现下面任一情况就需要回收：

- 玩家必须读完规则清单才知道按钮为什么存在
- 首屏最显眼的信息是价格、百分比、概率或限制条件
- 本应属于确认层的信息，占掉了正文主要面积
- 页面像帮助中心、系统公告或角色面板，而不像场景

## Border Audit

逐块看结构，确认装饰有且只有一个语义：

- 结构分隔
- 内容分组
- 交互反馈

发现下面任一情况就需要回收：

- 同一语义在相邻层级被重复绘制
- 激活态和结构分隔用了同一种强调语言
- 某个模块必须靠完整外框才能看懂层级
- 去掉一条线后信息不受损，说明那条线本来就多余

## Interaction Audit

检查交互是否安静一致：

- tab、切换器、筛选器的激活态是否低噪音
- 交互高亮是否比正文内容更抢眼
- 正文里是否混入了跨场景导航动作
- 当前任务按钮和全局导航按钮是否被清楚区分

## Shared-Component Audit

如果改动涉及共享组件，再问：

- 共享组件进入主流程正文后，是否仍保留不必要的展示冲动
- 是否通过新增壳层透传参数来解决局部视觉问题
- 是否应该把规则收进一个聚焦组件，而不是散落在多个调用点
- 是否不小心把主流程规则扩散成了全局规则

## Spot Checks

至少抽查这三类代表页各一个：

- 入口型页面
- 流程型页面
- 列表或服务型页面

如果本次改动涉及全局入口，还要补查：

- 顶部身份入口
- 底部全局导航
- 全局预览或提醒型入口

## Exit Criteria

只有同时满足这些条件，才算这次 UI 改动没有回退：

- 玩家第一眼能明确感知身份层、任务层、导航层
- 玩家不需要长时间阅读规则，也能理解当前场景的用途
- 正文里没有新的身份中心
- 正文标题只保留组织信息所需的最小集合
- 精确代价、风险和数值说明只在真正需要决策的地方出现
- 装饰没有重复表达同一语义
- 当前场景任务仍然是页面注意力中心
